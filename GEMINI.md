# Gemini CLI 설정 및 규칙

이 파일은 Gemini CLI의 동작 방식과 답변 스타일을 정의하는 설정 파일입니다. 이 프로젝트의 모든 작업은 아래 규칙을 최우선으로 따릅니다.

## 1. 사용자 중심 소통 (비전공자 맞춤형)
- **쉬운 설명**: 전문 용어보다는 누구나 이해할 수 있는 일상적인 용어를 사용하여 설명합니다.
- **상세한 주석**: 코드 변경 시, 각 부분이 어떤 역할을 하는지 한글로 상세히 주석을 답니다.
- **진행 상황 공유**: 현재 어떤 작업을 하고 있는지, 왜 이 작업을 하는지 단계별로 설명합니다.
- **결과 확인**: 작업 완료 후 사용자가 어떻게 확인하면 되는지 친절하게 안내합니다.

## 2. UI/UX 개선 최우선
- **직관적 디자인**: 사용자가 고민 없이 행동할 수 있는 직관적인 UI 구성을 지향합니다.
- **반응형 대응**: 모바일과 데스크톱 환경 모두에서 최적의 경험을 제공하도록 작업합니다.
- **일관성 유지**: 기존 디자인 시스템(shadcn/ui, Tailwind CSS)과 톤앤매너를 유지하면서 완성도를 높입니다.
- **피드백 강화**: 버튼 클릭, 로딩, 성공/실패 시의 시각적 피드백을 강화합니다.

## 3. 답변 품질 향상 가이드
- **핵심 요약**: 장황한 설명보다는 핵심적인 내용을 먼저 전달합니다.
- **시각적 요소 활용**: 필요한 경우 표나 코드 블록을 활용하여 정보를 구조화합니다.
- **능동적 제안**: 단순히 요청받은 것만 수행하는 것이 아니라, 더 나은 UX를 위한 개선 사항을 먼저 제안합니다.

## 4. 코딩 규칙 (추가)
- **한글 커밋 메시지**: 커밋 메시지는 반드시 한글로 작성합니다.
- **한 파일씩 수정**: 코드 안정성을 위해 한 번에 한 파일씩 수정하고 저장합니다.
- **서버 모니터링**: 수정 후 빌드 에러나 화면 깨짐 현상이 발생하는지 항상 확인합니다.
