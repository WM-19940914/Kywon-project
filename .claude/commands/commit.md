# /commit — 한글 커밋 자동 생성

변경사항을 분석하여 한글 커밋 메시지를 자동 생성하고 커밋한다.

## 절차

1. `git status`로 변경된 파일 확인
2. `git diff --staged`와 `git diff`로 변경 내용 분석
3. 최근 커밋 메시지 스타일 참고
4. 한글 커밋 메시지 생성
5. 사용자 확인 후 커밋 실행

## 커밋 메시지 규칙

- **언어**: 한국어만 사용
- **형식**: 기능명/개선사항을 간결하게 나열, 쉼표로 구분
- **예시**:
  - `에스원 정산관리: 정산월 DB 저장 버그 수정`
  - `발주폼 개선, 배송관리 수동 상태전환, 엑셀 붙여넣기`
  - `배송관리 페이지 재설계 및 사이드바 메뉴 개편, 빌드 에러 전체 수정`
- 영어, 이모지, conventional commit prefix(feat:, fix:) 사용 금지
- 너무 길지 않게 핵심만 (한 줄 70자 이내 권장)

## 주의사항

- `.env.local`, `.cursor/mcp.json` 등 민감 파일은 커밋 대상에서 제외
- 커밋 전 반드시 사용자에게 메시지 확인받기
- `--no-verify` 사용 금지
